2D Game Engine

A 2D Game Engine created in C++ as part of a university unit.

Project Overview

As part of a team project, I led the development of a custom UI system within a 2D game engine written in C++, using ImGui for the game UI. The goal was to create a robust and fully modular UI framework that allowed users to dynamically create and configure UI components at runtime.

My personal contributions were developing the factory class system capable of generating UI elements from user defined data. The system supports key UI components including labels, buttons, checkboxes and images, all of which can be modified in real time through Lua scripting. The scripting integration was successfully developed by working closely with the scripting team.

As the UI Team Lead, I coordinated development efforts between members of the sub team and talked with other teams to understand their wants and needs within the project and how the UI team can aid in helping. This involved active communication, iterative feedback cycles and including user testing.

After completeing the core UI framework, I transitioned into testing, where I created several gameplay prototypes including; a particle system, a Flappy Bird clone and a basic platformer. Each prototype was developed to test the many differnt systems developed for the engine from physics to UI. These tests provided valuable feedback to each team that led to new features and engine optimisation.

Features & Techniques

  • Architecture — patterns such as Singletons and Factory were used to create a modular and robust framework, making it simple to add new UI elements.
  • C++ — By creating a game engine, I gained a deeper understanding of the underlying architecture and programming techniques involved in developing complex software.
  • Testing — By conducting tests through small game prototypes, I strengthened my ability to evaluate system peformance, identify bugs and ensure features function as intended.
